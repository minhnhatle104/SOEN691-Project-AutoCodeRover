Deserialization issue with `@JsonIgnore` and `@JsonCreator` + `@JsonProperty` for same property name
Issue originally found on jackson-module-kotlin: https://github.com/FasterXML/jackson-module-kotlin/issues/124. Issue reproducible with Java code.

Entity class:
```java
static public class Foo {
  @JsonIgnore 
  public String query;

  @JsonCreator
  @ConstructorProperties("rawQuery")
  public Foo(@JsonProperty("query") String rawQuery) {
    query = rawQuery;
  }
}
```

Fails this test:
```java
Foo deserialized = new ObjectMapper().readValue("{\"query\": \"bar\"}", Foo.class);
assertEquals("bar", deserialized.query);
```
